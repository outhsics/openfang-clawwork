# Cleanup Memo Case

This is a concrete case-study folder showing what a "real office deliverable" can look like inside the current `summary-only` bridge mode.

## Source task

- Occupation: `Administrative Services Managers`
- Sector: `Government`
- Estimated task value: `$212.06`
- Source idea:
  Draft a professional memo explaining a rotating cleanup schedule and recreate the attached schedule into an editable Excel file for staff use and volunteer sharing.

## Files in this folder

- [task_stub.json](task_stub.json): task metadata formatted for OpenFang routing
- [delivery/cleanup_memo.md](delivery/cleanup_memo.md): sample memo deliverable
- [delivery/cleanup_schedule.csv](delivery/cleanup_schedule.csv): editable schedule artifact
- [workflow_run_prompt.txt](workflow_run_prompt.txt): one-shot input for OpenFang workflow tests

## Why this case matters

It is a good first proof point because it combines:

- a real document outcome
- a real spreadsheet-style artifact
- low ambiguity
- operational value that a small business or department would actually pay for

It is much closer to billable office work than a generic agent demo.
